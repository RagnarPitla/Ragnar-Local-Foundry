iWhat is Foundry Local

Foundry Local is Microsoft's end-to-end on-device AI runtime. It ships a lightweight (~20 MB) runtime built on ONNX Runtime, a curated catalog of quantized models, native SDKs (C#, JavaScript, Python, Rust), and automatic hardware acceleration.

🔒 Private

Prompts and outputs stay on the device.

✈️ Offline

Works with no connectivity after the first model download.

💸 No token cost

No cloud inference charges, no backend to maintain.

On Apple Silicon it accelerates inference with the WebGPU execution provider (Dawn targeting Metal) and falls back to CPU (MLAS). There is no CUDA or NPU path on Mac.

?Do I need Docker?

🐳
No. Docker is not required for Foundry Local on Mac.

Foundry Local runs models directly on ONNX Runtime, not in containers. The Ragnar-Local-Foundry repo has zero Docker references, and the prerequisites are just Homebrew + Xcode Command Line Tools. Docker Desktop can stay installed on your Mac, but you can safely ignore it here. You would only reach for Docker later if you wanted to add a browser chat UI such as Open WebUI on top of the local API.

▶Quick start (TL;DR)

Foundry Local is already installed on this machine, so you are three commands away from chatting with a local model.

Terminal · fastest path
# 1. Start the local model service
foundry service start

# 2. Download + run a tiny starter model (downloads on first use)
foundry model run qwen2.5-0.5b

# 3. Type a prompt at the interactive REPL, then /exit to quit
✅
Prefer the repo automation?

From the repo root you can run make demo to install, pull the starter profile, serve, and health-check end to end. Run make help to see every target.

2Install Foundry Local

Option A — repo automation (recommended)

Terminal · from the repo root
make install      # installs Foundry Local via Homebrew and starts the service
make models       # downloads the "starter" profile (or PROFILE=balanced / power)
make demo         # install + pull starter + serve + health check, end to end

Option B — manual install

Terminal
# 1. Add the Microsoft tap and install the runtime + CLI
brew tap microsoft/foundrylocal
brew trust microsoft/foundrylocal
brew install foundrylocal
foundry --version

# 2. See what your hardware can run (downloads execution providers on first run)
foundry model list

# 3. Run a model interactively (downloads on first use)
foundry model run qwen2.5-0.5b
⚠️
Homebrew 6.x gotcha: "Refusing to load formula from untrusted tap".

Newer Homebrew gates third-party taps behind an explicit trust step. If brew install foundrylocal is refused, run brew trust microsoft/foundrylocal first, then reinstall. The repo installer (scripts/install-mac.sh) does this automatically.

3Start the service & use it

Foundry Local runs a small background service that manages models and exposes a local endpoint. Start it once, then talk to a model.

  1. Start the service
    Run foundry service start. First launch may download execution providers, so give it a minute.
  2. Confirm it is healthy
    foundry service status should show a green running state and the dynamic endpoint it is listening on.
  3. Run a model
    foundry model run qwen2.5-0.5b opens an interactive REPL. The model downloads on first use, then is cached for offline runs.
  4. Chat, then exit
    Type prompts at the prompt. Use /exit (or Ctrl+C) to leave the REPL. The service keeps running in the background.
Terminal · service lifecycle
foundry service start        # start it
foundry service status       # is it up, and on which endpoint
foundry service restart      # fixes "Request to local service failed"
foundry service stop         # stop when you are done
🩺
One-shot sanity check.

From the repo root, make health verifies version, service status, catalog access, cache access, and endpoint discovery in a single pass.

4Pick a model

The catalog is dynamic and hardware-aware, so the authoritative list for your machine is always foundry model list. This repo groups a curated subset into three profiles:

5CLI cheat sheet

foundry · most-used commands
foundry model list                       # list models for your hardware
foundry model list --filter task=chat-completion
foundry model list --filter alias=qwen*
foundry model info qwen2.5-0.5b          # details (add --license for terms)
foundry model download phi-4-mini
foundry model run qwen2.5-0.5b           # interactive REPL

foundry service status                   # up? which endpoint?
foundry service restart                  # fix a stuck service
foundry service ps                       # running model processes
foundry service diag                     # diagnostics

foundry cache list                       # what is cached locally
foundry cache location                   # where models are stored
foundry cache remove <model>             # free disk space

6Use the OpenAI-compatible API

Foundry Local exposes an OpenAI-compatible server on a local, dynamic port. Start it, discover the endpoint, then point any OpenAI client at $ENDPOINT/v1 with a placeholder API key.

Terminal · serve + discover endpoint
make serve                   # or: bash scripts/serve.sh
foundry service status       # prints the dynamic endpoint
Terminal · call it with curl
curl "$ENDPOINT/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen2.5-0.5b","messages":[{"role":"user","content":"Hello"}]}'
🔑
The API key is a placeholder.

Local auth does not use a real key, but OpenAI SDKs require a value. Pass something like local-placeholder.

8Troubleshooting

SymptomLikely causeFix
Request to local service failedService stopped or unhealthy.foundry service restart, then foundry service status.
Model not foundAlias not available on this machine.foundry model list and choose an available alias.
First model run is slowModel + execution providers download on first use.Wait for the first run; later runs use the cache.
Low disk spaceCached models consume storage.foundry cache list then foundry cache remove <model>.
Refusing to load formula ... untrusted tapHomebrew gates third-party taps.brew trust microsoft/foundrylocal, then reinstall.
Cannot find REST endpointLocal server uses a dynamic port.foundry service status or make serve.
OpenAI SDK asks for an API keySDK requires a value even locally.Use a placeholder such as local-placeholder.
Slow responses on large modelsModel size / memory pressure / CPU fallback.Use a smaller alias or a lower profile.
